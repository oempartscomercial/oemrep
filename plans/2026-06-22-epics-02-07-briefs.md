## Épico 2 — Cadastros & Acesso

**Goal:** Multi-fábrica, clientes (multi-fábrica) e usuários com perfis e permissão
por fábrica; infra de auditoria pronta para uso transversal.

**RFs:** RF01, RF02 · infra RF32 · RNF de acesso/perfis (ADR-009).

**File map (previsto):**
- `src/domain/cadastro/cnpj.ts` (+test) — validação/normalização de CNPJ (função pura).
- `src/domain/auditoria/evento.ts` (+test) — montar `EventoAuditoria` (campo/antes/depois/usuário/ts).
- `prisma/schema.prisma` — `Fabrica`, `Cliente`, `ClienteFabrica` (N:N), `Usuario`,
  `UsuarioFabrica`, `EventoAuditoria`.
- `src/app/(app)/cadastros/...` — telas de Fábricas, Clientes, Usuários.
- `src/lib/authz.ts` (+test) — `podeAcessarFabrica(usuario, fabricaId)` (pura).

**Tarefas ordenadas (intent + foco de teste):**
1. Validador de CNPJ — TDD (válidos, inválidos, máscara). 
2. Schema Prisma de Fabrica/Cliente/N:N + migração — teste de criação/leitura.
3. Função de auditoria pura — TDD (gera evento com diffs corretos).
4. Tabela `EventoAuditoria` + helper `registrarAlteracao()` — teste de gravação imutável.
5. `Usuario`/perfis/`UsuarioFabrica` + `podeAcessarFabrica` — TDD da regra de acesso.
6. Telas CRUD de Fábricas/Clientes/Usuários com checagem de perfil — e2e mínimo.

**DoD:** cadastrar fábrica e cliente multi-fábrica; criar usuário e restringir por
fábrica; toda alteração grava EventoAuditoria; `npm test` verde.

## Épico 3 — Pedidos

**Goal:** Criar pedido (upload Excel + manual + `S/N`), itens com status e quantidades
derivadas, ciclo de vida, lista com filtros e tela de detalhe.

**RFs:** RF03, RF04, RF05, RF06, RF07, RF08(telas), RF10, RF11.

**File map (previsto):**
- `src/domain/pedido/estado.ts` — já existe (Épico 1); estender com
  `recalcularEstado(itens)` (TDD: COMPLETO inclui FORA_DE_FABRICACAO/DESISTENCIA, ADR-005).
- `src/domain/pedido/item.ts` (+test) — `qtdPendente`, snapshot `qtdPendenteCongelada`
  ao resolver por não-faturamento (ADR-008).
- `src/domain/importacao/excel.ts` (+test) — extrair itens de uma planilha (ExcelJS).
- `prisma/schema.prisma` — `Pedido`, `ItemPedido`.
- `src/app/(app)/pedidos/...` — lista (filtros Em andamento/Concluídos/Arquivados/Todos),
  novo (manual), importar (Excel), `[id]` detalhe (abas Itens/Notas/Histórico).

**Tarefas ordenadas:**
1. `recalcularEstado(itens)` — TDD das regras de COMPLETO (ADR-005).
2. Cálculo de `qtdPendente` + snapshot `qtdPendenteCongelada` — TDD (ADR-008).
3. Parser de Excel de pedido — TDD com planilha-exemplo (referência/qtd/valor).
4. Schema `Pedido`/`ItemPedido` + migração — teste de persistência.
5. Criar pedido manual + `S/N` (flag) — teste de serviço + e2e.
6. Importar pedido via Excel → revisar → confirmar — e2e.
7. Lista de pedidos com filtros segmentados — teste de filtro (puro) + e2e.
8. Detalhe do pedido (abas) com arquivamento reversível (RF10) — e2e.

**DoD:** importar/criar pedido, ver itens e ciclo de vida, arquivar e reexibir;
estados respeitam ADR-005/008; auditoria registrando; testes verdes.

## Épico 4 — Conferência de NFe

**Goal:** Importar XML, conferir item a item contra pendências, baixa parcial
progressiva, NFe cobrindo vários pedidos, relatório de cruzamento.

**RFs:** RF12, RF13, RF14, RF15, RF16, RF17, RF09.

**File map (previsto):**
- `src/domain/nfe/parser.ts` (+test) — fast-xml-parser → `{destinatarioCnpj, chave,
  itens[], totalProdutos, totalNota}`. Meta < 5 s.
- `src/domain/nfe/conferencia.ts` (+test) — casar por `CNPJ + ref + qtd + valorUnit`,
  retornar divergências (RN04). 
- `src/domain/nfe/baixa.ts` (+test) — distribuir quantidades por `ItemFaturado`,
  recalcular pendência (RF09/RN11).
- `prisma/schema.prisma` — `NotaFiscal`, `ItemFaturado`, `NotaFiscalPedido` (N:N).
- `src/app/(app)/conferencia/...` — upload XML, grade editável, confirmação de baixa.

**Tarefas ordenadas:**
1. Parser de NFe XML — TDD com XML real de exemplo (Bowden/Autoflex).
2. Conferência item a item + sinalização de divergência — TDD (casos OK/qtd/valor/CNPJ).
3. Baixa progressiva via ItemFaturado — TDD (parcial, múltiplas NFes por item).
4. Schema NFe/ItemFaturado/N:N + migração — teste de persistência.
5. NFe cobrindo vários pedidos do mesmo cliente — TDD do vínculo (RN10).
6. Tela de conferência editável (grade visual) — e2e do fluxo upload→baixa.
7. Relatório de cruzamento por NFe (substitui aba manual) — teste de geração (RF17).

**DoD:** upload de XML real conferido em ≤3 etapas; divergências destacadas; baixa
parcial correta; item atendido por várias NFes; auditoria; testes verdes.

**DoD:** abrir chamado da NFe com itens afetados; registrar eventos; crítico após 30
dias reaparece; testes verdes.

## Épico 7 — Análise, Alertas & Auditoria

**Goal:** Painel PEDIDOS×NFE com gap (só produtos), alerta de pedido sem NFe (7 dias),
dashboard, consulta de auditoria e exportação XLSX.

**RFs:** RF31, RF33, RF34 · consolidação de RF32.

**File map (previsto):**
- `src/domain/analise/gap.ts` (+test) — gap por mês/fábrica/cliente usando
  `totalProdutos` (ADR-007).
- `src/domain/alerta/semNfe.ts` (+test) — `pedidosSemNfeVencidos(pedidos, hoje, prazo=7)` (ADR-006).
- `src/domain/export/xlsx.ts` (+test) — gerar XLSX de qualquer listagem (ExcelJS).
- `src/app/(app)/pedidos-x-nfe/...`, `/alertas/...`, `/` (dashboard com KPIs).

**Tarefas ordenadas:**
1. Cálculo do gap (só produtos) — TDD (ADR-007).
2. Regra de alerta "pedido sem NFe" (7 dias) — TDD com datas (ADR-006).
3. Exportação XLSX genérica — TDD (cabeçalhos/linhas corretos) (RF33).
4. Painel PEDIDOS×NFE (gráfico+consolidado mensal) + export — e2e (RF31).
5. Central de alertas (sem NFe no MVP) — e2e (RF34).
6. Dashboard com KPIs + fila do dia — e2e.
7. Tela de consulta de auditoria (por período/usuário/registro) — e2e (RF32).

**DoD:** gap visível por fábrica/cliente; alerta de pedido sem NFe disparando aos 7
dias; exportar XLSX; consultar auditoria; testes verdes. **MVP completo.**
